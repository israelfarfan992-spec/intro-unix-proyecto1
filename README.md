1. Project Description
This project involves the deployment and security of a Debian 13, integrating multiple layers of cybersecurity. The work encompasses everything from protecting hardware and data at rest using disk encryption, and advanced volume management, to implementing best practices for credential management and secure communications.

2. Project Objectives
2.1. General Objective
To implement a secure workstation environment using open-source tools, applying cryptography concepts, volume management, and privacy standards to mitigate risks of unauthorized access, loss of data integrity, and identity theft.

2.2. Specific Objectives
Secure Infrastructure: Configure a Debian 13 system utilizing a Full Disk Encryption scheme over LVM, ensuring data remains inaccessible in the event of physical hardware loss.

Entropy-Based Management: Apply Shannon’s Entropy formula to determine the actual strength of passwords and utilize encrypted database managers to eliminate the use of weak or recycled credentials.

Integrity Validation: Demonstrate the avalanche effect in cryptographic hash functions and emphasize the importance of software verification to prevent supply chain attacks.

Security Auditing (Basic Pentesting): Evaluate the resistance of various entropy levels against dictionary and brute-force attacks using Hashcat, establishing a clear correlation between password complexity and computational time.

Identity and Privacy: Establish an asymmetrically encrypted communication channel and configure a Git development workflow where every commit is digitally signed, linking the author’s identity unequivocally through GPG.

Regulatory Compliance: Analyze and compare legal data protection frameworks under the European standard and the Ecuadorian Organic Law on Protection of Personal Data.
